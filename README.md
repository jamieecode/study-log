### 📝 소개 | Introduction
 
그동안 검색하고 지나쳤던 정보,  
그때그때 궁금해서 찾아봤던 내용,  
그리고 개발하면서 실제로 마주친 이슈들을  
기록으로 남기기 위해 시작한 개인 스터디 로그입니다.

매번 찾아보긴 했지만,  
그 순간에만 이해하고 시간이 지나면 다시 잊어버리는 경우가 많았습니다.  
“나중에 다시 보면 좋을 텐데”라는 아쉬움이 반복되었고,  
그 아쉬움을 줄이기 위해 기록을 남기기 시작했습니다.

완벽하게 정리된 문서보다는,  
관심을 가졌던 순간과 고민의 흐름을 남기는 것에 의미를 두고 있습니다.


This repository is a personal study log created to record  
things I’ve searched for, been curious about,  
and issues I’ve encountered while developing.

Although I often looked things up,  
most of that knowledge stayed only for the moment.  
Over time, I found myself searching for the same topics again.  
To avoid losing those learnings, I started writing them down.

Rather than perfectly polished documents,  
this repository focuses on capturing learning moments and problem-solving traces.

---

### 📂 Index

이 레포지토리는 아래와 같은 구조로 정리되어 있습니다.  
The repository is organized into the following directories.

#### 📘 `study/`

개발하면서 궁금해서 찾아본 내용,  
공식 문서, 아티클, 개념 정리 등  
개인 학습 기록을 정리하는 공간입니다.

This directory contains things I looked up while studying:  
concepts, documentation notes, articles, and learning summaries.

#### 🐞 `issue/`

개발 중 실제로 맞닥뜨린 이슈와 문제 상황,  
그리고 원인 분석과 해결 과정을 기록합니다.  

This directory logs real issues encountered during development,  
including problem descriptions, root causes, and how they were resolved.

#### 🏢 `domain/`

현재 재직 중인 회사의 도메인, 비즈니스 분야,  
업무와 관련된 개념 및 흐름을 이해하기 위해 정리한 공간입니다.

This directory contains notes related to the domain and business area  
of the company I currently work for,  
helping me better understand the context and background of the work.
