# CDN

![스크린샷 2025-12-16 100121.png](%EC%8A%A4%ED%81%AC%EB%A6%B0%EC%83%B7_2025-12-16_100121.png)

CDN(Content Delivery Network, 콘텐츠 전송 네트워크): 웹 콘텐츠를 세계 곳곳에 있는 여러 서버에 분산하여 저장하는 분산 서버 네트워크 시스템.

목적: 서버와 최종 사용자의 물리적 거리를 줄여서 웹 서비스의 성능, 속도, 신뢰성을 향상시키고 서버 부하를 최소화하는 것

트래픽이 많지 않고 국내 방문자만 있는 사이트는 도입할 필요가 없지만, 웹사이트 로딩 지연이 높거나 트래픽이 많고 서버 성능 이슈가 발견되면 고려해볼 필요 있음.

---

## 동작방법

- 오리진 서버: 웹사이트 원본 콘텐츠 저장하고 있는 중앙 서버. 웹사이트 운영자는 오리진 서버에서 콘텐츠를 업데이트하거나 삭제하고 오리진 서버를 관리해야함
- 엣지 서버: 전 세계 곳곳에 위치한 분산 서버. 오리진 서버의 콘텐츠를 캐싱하고 있음. CDN은 가장 가까운 엣지서버에서 사용자에게 콘텐츠를 제공하기에 사용자가 최신 콘텐츠를 빠르게 받을 수 있다.캐시 정책에 따라 콘텐츠를 업데이트하고, CDN 공급업체가 관리함.
- DNS 서버: 오리진 서버와 엣지 서버를 연결해주는 역할. 사용자가 오리진 서버에 요청을 보내면 DNS 서버는 사용자를 가장 가까운 엣지 서버로 라우팅 해줌.

사용자가 웹사이트에 접속했을 때

1. DNS서버는 사용자 위치를 기준으로 가장 가까운 엣지 서버(캐싱서버)의 IP 주소를 반환함
2. 엣지 서버는 요청된 콘텐츠가 캐싱되어있는지 확인. 엣지 서버에 해당 콘텐츠가 캐싱되어 있지 않거나 캐시된 콘텐츠가 만료되었다면 오리진 서버에서 최신 콘텐츠를 가져옴
3. 엣지 서버는 사용자에게 콘텐츠 제공함
