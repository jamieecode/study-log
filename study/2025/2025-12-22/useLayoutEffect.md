# useLayoutEffect

- useEffect와 가장 큰 차이는 실행 시점
    - **useEffect**
        - 렌더링 및 페인트 작업 후. 화면렌더링 후 비동기적으로 실행되어 성능에 유리
        - data fetch, event handler, state reset 등에 사용
        - DOM 조작시 사용하면 화면 깜빡임 발생
    - **useLayoutEffect**
        - 렌더링 후, 페인트 작업 전(레이아웃 계산 직후). 화면 렌더링 후 동기적으로 실행되어 DOM 조작시 발생하는 깜빡임을 방지하는데 사용됨
        - DOM의 레이아웃(크기, 위치 등)을 읽고 바로 수정해야 할 때(애니메이션, 툴팁이나 스크롤 위치 확인 등등)
        - ref가 업데이트 되기 전 이전 값 확인해야 할 때
        - 렌더링 파이프 라인을 막음. 반드시 필요할 때만 사용해야 하며 남용하면 성능 저하 유발
