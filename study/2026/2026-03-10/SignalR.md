# SignalR

Microsoft에서 제공하는 ASP.NET용 오픈소스 라이브러리로, 서버가 클라이언트에 실시간으로 데이터를 푸시(push)할 수 있도록 해준다. 기존 HTTP 요청/응답 방식과 달리, 클라이언트의 요청 없이도 서버가 즉시 데이터를 전달할 수 있다. 아래와 같은 서비스에 적합하다.

- 실시간 채팅
- 알림 시스템
- 실시간 대시보드
- 주식/환율 시세
- 스포츠 점수 업데이트
- 협업 도구 (문서 공동 편집 등)

### 핵심 특징

- 서버 → 클라이언트 실시간 푸시
- 양방향 통신 지원
- 자동 전송 방식 선택 (WebSockets 우선)
- 전송 폴백(Fallback) 지원
- RPC 스타일 API 제공
- 그룹 관리 및 브로드캐스트 지원
- ASP.NET Core와 자연스럽게 통합

### 작동 원리

SignalR은 전송 계층을 추상화하여, 가능한 최적의 전송 방식을 자동으로 선택한다.

### 지원 전송 방식

|  전송 기술 | 설명  |
| --- | --- |
| WebSockets  | 가장 성능이 좋고 양방향 통신 가능 |
| Server-Sent Events (SSE) | 서버 → 클라이언트 단방향 |
| Long Polling | 최후의 폴백 방식 |

→ WebSockets 사용이 불가능하면 자동으로 다른 방식으로 전환된다.

### Hub 구조

SignalR은 **Hub**를 중심으로 동작하는데, Hub는 서버와 클라이언트 간의 RPC 통신을 담당한다.

- 서버 → 클라이언트 메서드 호출 가능
- 클라이언트 → 서버 메서드 호출 가능
- 그룹 생성 및 브로드캐스트 지원

### WebSocket vs SignalR 차이

| 구분  | WebSocket  | SignalR  |
| --- | --- | --- |
| 종류  | 통신 프로토콜 | 실시간 통신 라이브러리 |
| 폴백 지원 | x | o |
| 그룹 관리 | x | o |
| 자동 재연결 | x | o |
| ASP.NET 통합 | x | o |

→ SignalR은 WebSocket 위에 구축된 고수준 추상화 라이브러리이다.

### 확장성 (Scale-Out)

단일 서버에서는 문제가 없지만, 여러 서버로 확장할 경우 메시지 동기화가 필요하다.

### 확장 방식

- Azure SignalR Service
- Redis Backplane
- SQL Server Backplane

→ 여러 서버 환경에서는 백플레인 구성이 필수이다.

### 장점

- 실시간 양방향 통신
- 전송 자동 선택 및 폴백
- RPC 스타일 통신
- 그룹/브로드캐스트 지원
- ASP.NET Core와 통합
- 확장 가능한 구조
