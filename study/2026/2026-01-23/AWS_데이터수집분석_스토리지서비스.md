# AWS 데이터 수집·분석 및 스토리지 서비스

## 전 세계에 분산된 데이터를 하나의 S3 버킷으로 빠르게 수집하는 방법

### S3 Transfer Acceleration (S3 TA)

- 전 세계에 분산된 **AWS Edge Location**을 활용하여 데이터를 S3 버킷으로 전송하는 기능
- 사용자는 가장 가까운 Edge Location으로 데이터를 업로드하고, 이후 구간은 **AWS의 최적화된 전용 고속 네트워크**를 통해 S3 버킷으로 전달
- 대륙 간 데이터 전송 시 일반 인터넷망을 사용하는 것보다 훨씬 빠름
- 운영 복잡성이 거의 없으며, 글로벌 사용자로부터 데이터를 수집해야 하는 경우에 적합

---

### AWS Snowball Edge

- AWS에서 제공하는 **물리적 스토리지 장치**를 배송받아 데이터를 저장한 후 다시 AWS로 반송하는 방식
- 네트워크 대역폭을 거의 사용하지 않음
- 인터넷 연결이 불안정하거나, **수 페타바이트(PB) 규모의 대량 데이터**를 이전해야 할 때 적합
- 온라인 전송이 현실적으로 불가능한 환경에서 사용됨

---

### 파라미터 및 설정 관리

### AWS Systems Manager Parameter Store

- 애플리케이션 설정 및 민감한 값을 중앙에서 관리할 수 있는 서비스
- **파라미터 자동 교체(rotation) 기능을 기본적으로 제공하지 않음**
- 자동 교체가 필요할 경우 Lambda를 직접 구현하여 연동해야 함
- Secrets Manager와 비교 시 비용은 저렴하지만 기능은 제한적

---

### S3에 저장된 로그 데이터 분석

### Amazon Athena

- S3에 저장된 데이터를 **ETL 없이 그대로 분석**할 수 있는 서버리스 대화형 쿼리 서비스
- 표준 SQL을 사용하여 JSON, CSV, Parquet 등 다양한 형식의 데이터를 분석할 수 있음
- 서버를 프로비저닝하거나 관리할 필요가 없어 **운영 오버헤드가 매우 낮음**
- 쿼리를 실행할 때만 비용이 발생하는 **On-demand 방식**
- 간헐적인 로그 분석이나 단순 쿼리 작업에 최적

---

### Amazon Redshift

- 데이터를 Redshift 클러스터로 **사전에 로드**해야 함
- 데이터 웨어하우스 클러스터를 직접 운영해야 하므로 운영 복잡성이 높음
- 대규모 정형 데이터 분석과 복잡한 BI 쿼리에 적합

---

### Amazon EMR

- Spark, Hadoop 기반의 **대규모 빅데이터 처리 플랫폼**
- 복잡한 데이터 처리와 대용량 배치 작업에 강력
- 클러스터 관리가 필요하므로 운영 부담이 존재

---

### 공유 파일 시스템

### Amazon EFS (Elastic File System)

- 여러 가용 영역(AZ)에 있는 **수천 개의 EC2 인스턴스가 동시에 접근 가능한 네트워크 파일 시스템**
- 모든 인스턴스가 동일한 파일 시스템을 바라보므로, 어느 서버에 접속하더라도 동일한 파일과 문서를 확인할 수 있음
- 공유 스토리지가 필요한 웹 서버, CMS, 협업 환경에 적합
