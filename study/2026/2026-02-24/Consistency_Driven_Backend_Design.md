# Consistency-Driven Backend Design

## 정합성

- 논리적으로 모순되지 않음
- 도메인 규칙을 항상 만족
- DB 제약 조건을 항상 만족

## 트랜잭션

- 여러 단계의 작업을 하나의 묶음으로 처리하여, **전부 성공하거나 아니면 아예 아무것도 안 한 상태로 되돌리는 것.** 트랜잭션의 핵심 목적은 “**중간 상태가 외부에 노출되지 않도록 보장하는 것”**
- A가 B에게 1만 원을 송금하는 상황을 가정해보면
    1. **A의 계좌**에서 1만 원을 뺀다.
    2. **B의 계좌**에 1만 원을 더한다.
    
    만약 1번은 성공했는데 서버 장애로 2번이 실패한다면, A의 돈 1만 원은 사라지고 B는 받지 못한 상태가 되는데, 트랜잭션은 이런 문제를 방지하기 위해 1번과 2번을 하나로 묶어, 하나라도 실패하면 전체를 취소(Rollback)하고 모두 성공했을 때만 최종 반영(Commit)하도록 한다. 
    
- 실무에서 항상 고민해야 할 질문
    - 이 작업이 중간에 실패하면 어떤 데이터가 남는가?
    - rollback 되지 않으면 어떤 문제가 발생하는가?

## ACID

### Atomicity(원자성)

→ 전부 성공하거나 전부 실패

### Consistency(일관성)

→ 트랜잭션 실행 전후의 데이터베이스는 항상 정의된 규칙(데이터 타입, 제약 조건 등)을 만족하는 유효한 상태여야 함

### Isolation(격리성/고립성)

→ 동시에 실행되는 트랜잭션들이 서로의 중간 상태를 보지 못하도록 격리하는 성질

격리 수준(Isolation Level)에 따라 다음 문제가 발생할 수 있다

- Dirty Read
- Non-repeatable Read
- Phantom Read
- Lost Update

→ 대부분의 RDBMS는 기본적으로 완전한 격리가 아니라 타협된 격리 수준을 사용한다.

### Durability(지속성/영속성)

→ 성공적으로 완료된 트랜잭션의 결과는 시스템 장애가 발생하더라도 영구적으로 데이터베이스에 기록되어야 함

## DB 제약

- UNIQUE, FK, CHECK 제약은 반드시 DB 레벨에서 설정해야 함
- 코드 레벨 검증은 동시성 환경에서 race condition을 막지 못할 수 있다.
따라서 무결성 보장은 반드시 DB 제약과 함께 설계해야 한다.

---

## Race Condition

실행 순서에 따라 결과가 달라지는 문제

ex.  동시성 고려가 없다면

계좌 잔액 1000원 → 둘다 잔액이 충분하다는 걸 확인한 후 800원 출금 → -600원

### 해결 방법

- DB 레벨 락
    - SELECT FOR UPDATE
    - 비관적 락
- Optimistic Lock
    - version 컬럼 사용
    - 수정시 version 비교
- 원자적 쿼리

```sql
UPDATE account
SET balance = balance -800
WHERE balance >= 800
```

조건을 쿼리 안에 넣음. 하나의 원자적 연산으로 실행되므로, 잔액 조건 검증과 차감이 분리되지 않아 race condition을 방지할 수 있다.

## 계층 책임

- 계층을 분리하는 이유:
    - 책임을 명확히 하여 변경 영향 범위를 줄이기 위함
    - 테스트 가능성을 높이기 위함
    - 로직 중복을 방지하기 위함
    - 비즈니스 규칙이 HTTP나 DB 세부 구현에 의존하지 않도록 하기 위함
- 비즈니스 규칙은 Controller나 Repository가 아닌 Service(또는 Domain Layer)에 위치해야 한다.
- 일반적으로,
    - Controller → HTTP 처리
    - Service → 비즈니스 규칙
    - Repository → 데이터 접근
    - DB → 무결성 보장

## 백엔드 설계 체크리스트

- 이 API가 동시에 100번 호출되면 안전한가?
- 이 작업이 중간에 실패하면 데이터는 어떤 상태로 남는가?
- 코드 검증 없이 DB 제약만으로 막을 수 있는 부분은 없는가?
- 상태 전이가 정의되지 않은 경로는 없는가?
- 권한 검증이 조회/수정 모두에 적용되어 있는가?
