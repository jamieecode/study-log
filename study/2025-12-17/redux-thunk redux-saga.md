# redux-thunk/redux-saga

리덕스에서 비동기 작업을 돕는 미들웨어.

## redux-thunk

액션 객체가 아닌 함수를 디스패치. 함수를 디스패치할 때는 해당 함수에서 dispatch와 getState를 파라미터로 받아와야하고 이 함수를 만들어주는 함수를 thunk라고 부름. 

문제점은 action에서 너무 많은 일을함. 액션 생성자는 type과 payload가 담긴 객체를 생성해서 반환하는 역할을 수행하기로 했는데, thunk에서는 API요청이나 비동기 처리를 하여 본래의 역할이 모호해진다(어떤 경우는 객체, 어떤 경우는 함수를 반환)

## redux-saga

액션을 모니터링하고 있다가 특정 액션(JS 실행, 다른 액션 디스패치, 현재 상태 불러오는것)이 발생했을 때 미리 정해둔 로직에 따라 특정 작업이 이루어지는 액션에 대한 리스너. 

thunk에서 못하는 작업들을 할 수 있다.

1. 비동기 작업을 할 때 기존 요청을 취소할 수 있다.
2. 특정 액션이 발생했을 때 이에 따라 다른 액션이 디스패치되도록 하거나, 자바스크립트 코드 실행
3. 웹 소켓을 사용하는 경우 Channel 기능을 사용해 더욱 효율적으로 코드 관리 가능
4. API 요청 실패시 재요청 작업 가능.

## effects

### all

제너레이터 함수들이 들어있는 배열을 인자로 받음. 들어온 제너레이터 함수들은 병렬적으로 기능을 수행, 이 함수들이 모두 resolve 될 때까지 기다림(Promise.all과 유사)

### call

함수를 실행시키는 effect. 첫번째 인자에는 함수를 넣고 optional로 나머지 인자에 해당 함수에 넣을 인자를 넣을 수 있다. call 과 유사

### fork

fork는 함수를 비동기로 실행, call은 함수를 동기 실행. 순차적으로 함수가 실행되어야하는 api 요청 함수에서는 call 사용하면 됨.

### put

특정 액션을 dispatch 하는 effect. 

### takeEvery/takeLatest

인자로 들어온 액션에 대해 특정 로직을 실행시켜주는 effect. takeEvery는 인자로 들어온 모든 액션에 대해 로직을 실행시켜주는 반면, takeLatest는 기존에 실행중이던 작업이 있을 경우 이를 취소하고 가장 마지막으로 실행된 작업만 실행.
